Overview

Starting Pigale

The environement variable TGF should point to a repertory tgf which contains graphs coded either as tgf files or ascii files. When Pigale is started,the file a.tgf is opened for reading and writing. And the first graph is loaded and displayed.
While in the graph editor,the arrows of the toolbar let you load the next or previous graph of the current tgf file:
On the lowest band of the main window is displayed the current tgf input and output file.
In Pigale there always one active graph which is displayed in the Graph editor window. That graph can be saved, modified or displayed by different algoritms. A copy is created for any display not in the Graph editor window.

Windows

Under the menus, there is a toolbar that you can move around or hide.
Notice that some menus are not availaible if the current graph does fulfill the right properties for a given algorithm.
If you see a small manual " " at the left of a menu item, you can get a small help using the WhatisThis button
The windows next, to the Graph editor window, are used to show different views of a graph and these few notes.
The top right window is for messages (properties of the graph, warnings,etc). Just under are displayed the main properties of the graph, all of them are maintained in real time.

Pigale settings

Among other things, the Settings menu allows you to:
  • be in debug mode to increase the verbosity of Pigale and print more messages in log.txt
  • enable or disable the undo feature
  • define the Pigale colors used by Pigale. These colors are save in a file settings.txt. To come back to the original colors, you only need to erase that file
  • define the limits of pigale: i.e. define the maximum number of vertices above which graph are not displayed and slow algorithms are not performed.
  • define what the vertex show i.e. their labels or their index (or whateve the user programmed).
  • define some embed options, now for Schnyder drawings.
  • define the distance used to embed the grah in Rn-1
The dashed line at the top of the settings menu, is used to create a floating settings menu.

Using the editor

Mouse Actions

By default, the left button of the mouse is use to move vertices.
Other actions can be selected, by clicking on the radion buttons (Left Button).
If the shift is pressed, the actions are toggled the following way:
  • Add \ Del
  • Move a vertex \ Move all vertices of a given colour
  • Bissection \ Contraction
Other actions:
  • Add + control lef button Duplicate the coloured subgraph
  • Add + shift control lef button Duplicate the coloured subgraph and add edges between corresponding vertices
  • The middle button is set to move vertices.
  • The right button is set to display some information.

Colours and Thickness

The 16 upper small coloured squares at the right of the editing window, defines the brush color which appears whith a bigger square.
The brush color defines the color of the vertices when they are created, recolored or define the ones to be deleted.
The 16 lower sqares have the same functionnality for edges.
The last 3 squares are used to define the thickness of the edges.

Drawing on a grid

The grid size is defined by the slider position.
When a graph is loaded, Pigale computes the minimum size on which the graph can be drawn, without moving the vertices. When you click on Force G, Pigale will force all the vertices to fall on grid points, checking overlapping of vertices and checking, only if the graph is connected, that the genus did change. The undo G button, let you come back to the previous state. When you click on Fit grid, you are only enable to move or create vertices on grid points.

Undo/Redo

If this function is enable, a copy of the current graph is appended each time a program (except the editor) adds or delete edges. All the copies are destroyed when a new graph is loaded.
The save icon let you manually append a copy.
Undo Save Redo
There is no limit of the number of copies which are placed in the tgf file undo.tgf.

Embedding a graph in space

The axes X,Y,Z are respectively colored Red,Green,Blue

Mouse Action

  • Left button:
    • Moving along X axis -> Rotate along Y
    • Moving along Y axis -> Rotate along X
    • Double Click -> Start/Stop Rotate (depends on vertical slider)
  • Right button: Moving along X axis -> Rotate along Z
  • Middle Button: Translate
  • Wheel: Zoom

Buttons

The rightmost spin boxes let you select the projection to display. The facet button triangulate the graph, if it is planar, and draw the facets.
If you have no mouse wheel, you can use the keyboard up and down arrows to zoom, provided you have clicked on the drawing.

Tgf and ASCII files

We use a general proprietary format TGF.
A TGF file contains records, here corresponding to graphs. Each record consisting of a variable number of fields.
One of its main advantage is that we can write and read any complex data structure. But it is dependent of the processor type (Low or High indian etc).

We use very simple ASCII files to store graphs.
For example, the following file defines a graph called Triangle with three vertices labelled 10,20,30 and three edges. The last line indicates the end of file.
Triangle
10 20
20 30
30 10
0 0
